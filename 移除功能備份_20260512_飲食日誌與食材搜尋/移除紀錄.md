# 移除功能紀錄：飲食日誌 + 食材搜尋（Advisor 工具）

日期：2026-05-12
操作者：使用者指示 + Claude 執行

---

## 移除原因

使用者已從前端移除「飲食日誌」(`/diet`) 和「食材搜尋」(`/food-search`) 兩個獨立頁面。
為保持一致性，需同步移除 AI 營養顧問 (advisor.py) 中對應的 5 個工具，
避免 AI 呼叫已無前端渲染器的工具。

---

## 移除的 5 個 Advisor 工具

| 工具名 | 功能 | 對應意圖 | 後端 API |
|--------|------|----------|----------|
| `add_diet_log` | 新增飲食記錄 | `record_diet` | `POST /diet/logs` |
| `get_diet_logs` | 查詢飲食記錄 | `get_diet_logs` | `GET /diet/logs` |
| `update_diet_log` | 修改飲食記錄 | `update_diet_log` | `PUT /diet/logs/{id}` |
| `delete_diet_log` | 刪除飲食記錄 | `delete_diet_log` | `DELETE /diet/logs/{id}` |
| `search_food` | 搜尋食物營養 | `search_food` | `GET /food/search` |

1. **PROMPT_ECHO** — 移除飲食記錄/食物搜尋的範例

### 分發 / 驗證 / 生成（5 處）

## 不移除的部分（避免影響其他功能）

| 保留項目 | 原因 |
|----------|------|
| 後端 `diet.py` router | 行事曆、DRI 自動取飲食記錄等功能仍依賴 `/diet/logs` API |
| 後端 `food.py` router | 整合分析、食譜分析、營養素排名等頁面仍使用 `/food/search` |
| `_tool_query_dri` 內的自動取飲食記錄邏輯 | DRI 分析自動算今日攝取量，不是獨立工具 |
| 前端 `dietApi.js` / `foodApi.js` | 其他頁面（如 UnifiedAnalysisPage）可能仍 import |
| `food_name` 參數提取邏輯 | 其他意圖（如 search_symptom collecting）也可能需要 |
